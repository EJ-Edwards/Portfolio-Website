THM: Smol

A walkthrough of the TryHackMe Smol machine covering WordPress exploitation, vulnerable plugin abuse, backdoored plugin analysis, and privilege escalation.

TryHackMe WordPress Plugin Exploitation Privilege Escalation

Overview

Smol is a TryHackMe machine centred on a WordPress website — a common target due to its extensive plugin ecosystem. The box showcases a publicly known vulnerable plugin, highlighting the risks of neglecting software updates and security patches. Enhancing the learning experience, Smol introduces a backdoored plugin, emphasizing the significance of meticulous code inspection before integrating third-party components.

Reconnaissance

Nmap Scan

Step 1 — Run an initial Nmap scan to discover open ports and services:

nmap -sC -sV -oN nmap/smol.txt <TARGET_IP>

Findings:

  • Port 22 — SSH (OpenSSH)
  • Port 80 — HTTP (Apache / WordPress)

WordPress Enumeration

Step 2 — Enumerate the WordPress site with wpscan to identify installed plugins, themes, and users:

Tools Used

  • Nmap — port scanning and service detection.
  • WPScan — WordPress enumeration.
  • Searchsploit — exploit database lookup.
  • cURL / Burp Suite — HTTP request crafting.
  • Netcat — reverse shell listener.
  • LinPEAS / manual enumeration — privilege escalation.

Lessons Learned

  • Always keep WordPress plugins up to date — known vulnerabilities are trivially exploitable.
  • Audit third-party plugin source code before deployment; backdoors can be hidden in plain sight.
  • Principle of least privilege limits lateral movement even after initial compromise.
